# Техническое задание: TODO Manager

## 1. Общие сведения
**TODO Manager** — это веб-приложение для управления личными задачами, позволяющее пользователям создавать, классифицировать и отслеживать выполнение дел.

## 2. Бизнес-правила
1. **Авторизация:** Доступ к задачам осуществляется только после регистрации и входа в систему.
2. **Изоляция данных:** Пользователь видит только свои задачи.
3. **Статус задачи:** Задача может находиться в одном из трех состояний: `Новая`, `В процессе`, `Выполнена`.
4. **Приоритеты:** Каждая задача имеет уровень приоритета: `Низкий`, `Средний`, `Высокий`.
5. **Удаление:** Удаление задачи является безвозвратным.

## 3. Функциональные требования (ФТ)
*   **ФТ-1:** Регистрация и аутентификация пользователей (JWT).
*   **ФТ-2:** Создание задачи (название, описание, дедлайн, приоритет).
*   **ФТ-3:** Просмотр списка задач с возможностью фильтрации по статусу и приоритету.
*   **ФТ-4:** Редактирование и изменение статуса задачи.
*   **ФТ-5:** Удаление задачи.
*   **ФТ-6:** Поиск задач по ключевым словам в названии.

## 4. Нефункциональные требования (НФТ)
*   **НФТ-1 (Производительность):** Время отклика API не должно превышать 200 мс при стандартной нагрузке.
*   **НФТ-2 (Безопасность):** Пароли пользователей должны храниться в зашифрованном виде (bcrypt).
*   **НФТ-3 (Масштабируемость):** Архитектура должна поддерживать до 10,000 активных пользователей.
*   **НФТ-4 (Доступность):** Приложение должно быть доступно 99.9% времени (Uptime).
*   **НФТ-5 (Уведомления):** Система должна гарантировать доставку критических напоминаний в течение 30 секунд.
*   **НФТ-6 (Персонализация):** Алгоритм машинного обучения должен адаптироваться к паттернам пользователя за 14 дней использования.
*   **НФТ-7 (Надежность):** Система должна иметь fallback-механизмы при недоступности основных каналов уведомлений.

## 5. Аналитика и метрики
Для оценки эффективности продукта будут отслеживаться следующие показатели:
1. **DAU (Daily Active Users):** Количество уникальных пользователей в день.
2. **Task Completion Rate:** Процент задач, переведенных в статус «Выполнена» от общего числа созданных.
3. **Retention Rate:** Процент пользователей, возвращающихся в приложение через 7 дней после регистрации.
4. **Notification Effectiveness Rate:** Процент задач, выполненных после отправки напоминания.
5. **Average Response Time:** Среднее время между отправкой напоминания и выполнением действия.
6. **Snooze Rate:** Процент напоминаний, которые были отложены пользователями.
7. **Channel Performance:** Эффективность различных каналов уведомлений (email vs push vs SMS).
8. **Optimal Timing Score:** Метрика качества определения оптимального времени для напоминаний.

## 6. Дополнительная функциональность: Система совместной работы (Команды)
Для повышения ценности продукта вводится функционал **Командных проектов**:
*   **ФТ-7:** Создание рабочих пространств (проектов) и приглашение других пользователей по email.
*   **ФТ-8:** Назначение исполнителей на конкретные задачи внутри проекта.
*   **ФТ-9:** Ролевая модель доступа: `Владелец` (полный доступ), `Редактор` (изменение задач), `Наблюдатель` (только чтение).

## 8. Умная система напоминаний и уведомлений
Для повышения продуктивности пользователей и своевременного выполнения задач вводится интеллектуальная система уведомлений:

*   **ФТ-10:** Создание персонализированных напоминаний с настраиваемыми интервалами времени.
*   **ФТ-11:** Автоматические напоминания о приближающихся дедлайнах на основе приоритета задач.
*   **ФТ-12:** Отправка уведомлений через несколько каналов: email, push-уведомления, SMS (опционально).
*   **ФТ-13:** Умная система снуз-функции с адаптивными интервалами на основе поведения пользователя.
*   **ФТ-14:** Аналитика эффективности напоминаний и рекомендации по оптимизации расписания.
*   **ФТ-15:** Групповые уведомления для командных проектов с эскалацией при просрочке.

### 8.1. Типы напоминаний
1. **Статические:** Фиксированное время напоминания (например, каждый день в 09:00).
2. **Относительные:** Напоминание за определенное время до дедлайна (1 час, 1 день, 1 неделя).
3. **Интеллектуальные:** Система анализирует паттерны активности пользователя и предлагает оптимальное время.
4. **Эскалационные:** Повторные напоминания с увеличивающейся частотой для просроченных задач.

### 8.2. Бизнес-логика
1. **Приоритизация:** Высокоприоритетные задачи получают более частые и настойчивые напоминания.
2. **Адаптивность:** Система учится на основе действий пользователя (откладывание, игнорирование, выполнение).
3. **Тихие часы:** Автоматическое определение периодов, когда пользователь не хочет получать уведомления.
4. **Батчинг:** Группировка нескольких напоминаний в одно сводное уведомление для снижения навязчивости.

### 8.3. Входные данные

**Создание напоминания (POST /api/reminders)**
```json
{
  "task_id": "uuid",
  "reminder_type": "relative|static|smart",
  "trigger_time": "2024-01-15T09:00:00Z", // для static
  "offset_minutes": 60, // для relative (за час до дедлайна)
  "channels": ["email", "push", "sms"],
  "is_recurring": true,
  "recurrence_pattern": "daily|weekly|monthly",
  "snooze_enabled": true,
  "max_attempts": 3
}
```

**Настройки пользователя (PUT /api/users/notification-settings)**
```json
{
  "quiet_hours": {
    "start": "22:00",
    "end": "08:00",
    "timezone": "UTC+3"
  },
  "preferred_channels": ["email", "push"],
  "smart_scheduling_enabled": true,
  "batch_notifications": true,
  "escalation_enabled": true
}
```

## 7. Модель данных

### 7.1. Пользователь (User)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | UUID | Уникальный идентификатор пользователя |
| `email` | String | Электронная почта (уникальная) |
| `password_hash` | String | Хэш пароля (bcrypt) |
| `created_at` | DateTime | Дата регистрации |

### 7.2. Задача (Task)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | UUID | Уникальный идентификатор задачи |
| `user_id` | UUID | Идентификатор создателя (FK на User.id) |
| `project_id` | UUID | Идентификатор проекта (FK на Project.id, опционально) |
| `assignee_id` | UUID | Идентификатор исполнителя (FK на User.id, опционально) |
| `title` | String | Название задачи |
| `description` | Text | Подробное описание задачи |
| `status` | Enum | Статус: `Новая`, `В процессе`, `Выполнена` |
| `priority` | Enum | Приоритет: `Низкий`, `Средний`, `Высокий` |
| `due_date` | DateTime | Дедлайн (опционально) |
| `created_at` | DateTime | Дата создания |

### 7.3. Проект (Project)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | UUID | Уникальный идентификатор проекта |
| `name` | String | Название проекта |
| `owner_id` | UUID | Владелец проекта (FK на User.id) |

### 7.4. Участники проекта (ProjectMember)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `project_id` | UUID | FK на Project.id |
| `user_id` | UUID | FK на User.id |
| `role` | Enum | Роль: `Владелец`, `Редактор`, `Наблюдатель` |

### 7.5. Напоминание (Reminder)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | UUID | Уникальный идентификатор напоминания |
| `task_id` | UUID | FK на Task.id |
| `user_id` | UUID | FK на User.id |
| `reminder_type` | Enum | Тип: `Статическое`, `Относительное`, `Умное`, `Эскалационное` |
| `trigger_time` | DateTime | Время срабатывания напоминания |
| `offset_minutes` | Integer | Смещение в минутах от дедлайна (для относительных) |
| `channels` | JSON | Массив каналов доставки |
| `status` | Enum | Статус: `Активное`, `Отложено`, `Выполнено`, `Отменено` |
| `attempts_count` | Integer | Количество попыток отправки |
| `max_attempts` | Integer | Максимальное количество попыток |
| `last_sent_at` | DateTime | Время последней отправки |
| `snooze_until` | DateTime | Отложить до (для snooze-функции) |
| `effectiveness_score` | Decimal | Оценка эффективности (0-10) |
| `created_at` | DateTime | Дата создания |

### 7.6. Настройки уведомлений (NotificationSettings)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `user_id` | UUID | FK на User.id (Primary Key) |
| `quiet_hours_start` | Time | Начало тихих часов |
| `quiet_hours_end` | Time | Конец тихих часов |
| `timezone` | String | Часовой пояс пользователя |
| `preferred_channels` | JSON | Предпочитаемые каналы уведомлений |
| `smart_scheduling_enabled` | Boolean | Включено ли умное планирование |
| `batch_notifications` | Boolean | Группировка уведомлений |
| `escalation_enabled` | Boolean | Включена ли эскалация |
| `updated_at` | DateTime | Дата последнего обновления |

### 7.7. Лог уведомлений (NotificationLog)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | UUID | Уникальный идентификатор записи |
| `reminder_id` | UUID | FK на Reminder.id |
| `channel` | Enum | Канал: `Email`, `Push`, `SMS` |
| `status` | Enum | Статус: `Отправлено`, `Доставлено`, `Ошибка`, `Открыто`, `Нажато` |
| `sent_at` | DateTime | Время отправки |
| `delivered_at` | DateTime | Время доставки |
| `opened_at` | DateTime | Время открытия |
| `clicked_at` | DateTime | Время клика |
| `error_message` | Text | Сообщение об ошибке (если есть) |

### 7.8. Паттерны активности (UserActivityPattern)
| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `user_id` | UUID | FK на User.id |
| `hour_of_day` | Integer | Час дня (0-23) |
| `day_of_week` | Integer | День недели (1-7) |
| `activity_score` | Decimal | Оценка активности (0-1) |
| `completion_rate` | Decimal | Процент выполнения задач |
| `response_time_avg` | Integer | Среднее время отклика в минутах |
| `last_updated` | DateTime | Дата последнего обновления |

### 7.9. Связи
*   **Один-ко-многим:** Один `Пользователь` может быть владельцем множества `Проектов`.
*   **Многие-ко-многим:** `Пользователи` могут участвовать во многих `Проектах` через таблицу `ProjectMember`.
*   **Иерархия:** `Задачи` могут принадлежать `Проекту` или быть личными (при `project_id = NULL`).
